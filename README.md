# goit-algo-hw-04
Під час виконання завдання 3 було проведено порівняння трьох методів сортування:
1) Insertion Sort
2) Merge Sort
3) Timsort

В ході виконання коду було отримано такі результати для 10, 100, 1000 елементів.

|     Алгоритм   | 10 елементів | 100 елементів | 1000 елементів |
| Insertion Sort | 0.00000      | 0.00021       | 0.01501        |
|    Merge Sort  | 0.00001      | 0.00015       | 0.00129        |
|     Timsort    | 0.00000      | 0.00000       | 0.00007        |

За аналізами результатів:
Insertion Sort (Сортування вставками):
- Показує прийнятні результати лише на дуже малих наборах даних (до 10 елементів).
- Зі зростанням масиву час виконання збільшується значно швидше за інші алгоритми.
  Для масиву з 1000 елементів він у 11 разів повільніший за Merge Sort і більш ніж у 200 разів повільніший за Timsort.
- Це підтверджує його теоретичну складність O(n^2).
Merge Sort (Сортування злиттям):
- Починаючи з розміру масиву в 100 елементів, Merge Sort стає ефективнішим за сортування вставками.
- Алгоритм демонструє стабільну логарифмічну складність O(n log n).
  Хоча він значно швидший за вставки на великих даних, він все одно поступається вбудованому Timsort.
Timsort (Вбудований алгоритм Python):
- Виявився найбільш ефективним у всіх проведених тестах.
- Навіть при збільшенні масиву до 1000 елементів,
  Timsort демонструє неймовірно низький час виконання (0.00007 сек), що робить його абсолютним лідером.

У підсумок емпірична перевірка підтвердила, що поєднання сортування злиттям та вставками робить алгоритм Timsort надзвичайно
ефективним для широкого спектру даних.
